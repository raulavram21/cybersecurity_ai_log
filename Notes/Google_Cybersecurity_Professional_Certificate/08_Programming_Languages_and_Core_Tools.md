Programming Languages and Core Tools


Programming and Scripting

    Programming: A process used to create a specific set of instructions for a computer to execute tasks. (Context: Enables security analysts to complete repetitive processes with a high degree of accuracy and efficiency.)

    Python: A high-level programming language frequently used by security professionals to execute automation. (Context: Automates time-consuming tasks to reduce manual effort and mitigate the risk of human error.)

    Structured Query Language (SQL): A programming language designed to create, interact with, and request information from a database. (Context: Allows analysts to efficiently filter through millions of data points to retrieve the exact specific information needed during an investigation.)

Operating Systems

    Linux: An open-source operating system where the source code is publicly available for use and contribution. (Context: Unlike Windows or macOS, Linux heavily relies on a text-based Command-Line Interface (CLI). Analysts frequently use Linux commands to review error logs and investigate uncommon network traffic.)

Core Security Concepts & Tools

    Web Vulnerability: A unique flaw or bug in a web application that a threat actor could exploit using malicious code. (Context: Exploits lead to unauthorized access, data theft, or malware deployment. Security teams monitor the OWASP Top 10 to stay updated on the most critical web risks.)

    Antivirus Software (Anti-malware): A software program used to prevent, detect, and eliminate malware and viruses. (Context: It proactively scans device memory to find specific patterns that indicate the presence of a malicious payload.)

    Intrusion Detection System (IDS): An application that monitors system activity and generates alerts for possible intrusions. (Context: It continuously scans and analyzes network packets—small amounts of data moving through a network—to identify potential threats and unauthorized access attempts.)

    Encryption: The process of converting readable data (plaintext) into an unreadable, secure format (ciphertext). (Context: The primary goal of encryption is ensuring the Confidentiality of private data. It is fundamentally different from 'encoding', which uses a public algorithm simply to translate data formats so different systems can communicate.)

    Penetration Testing (Pen Testing): The act of participating in an authorized, simulated cyberattack. (Context: A highly proactive risk assessment technique used to identify external and internal vulnerabilities in systems, websites, and applications before real threat actors can exploit them.)
