Master Glossary A to Z


Section A

    Absolute file path: The full file path, which starts from the root. (Context: Essential for navigating Linux directories accurately.)

    Access controls: Security controls that manage access, authorization, and accountability of information. (Context: Primary mechanism for enforcing Confidentiality.)

    Active packet sniffing: A type of attack where data packets are manipulated in transit. (Context: Used to intercept/alter unencrypted traffic.)

    Address Resolution Protocol (ARP): A network protocol used to determine the MAC address of the next router or device on the path. (Context: Frequently targeted by ARP spoofing.)

    Advanced persistent threat (APT): An instance when a threat actor maintains unauthorized access to a system for an extended period of time. (Context: Highly skilled attackers, often state-sponsored, targeting critical infrastructure or IP.)

    Adversarial artificial intelligence (AI): A technique that manipulates artificial intelligence (AI) and machine learning (ML) technology to conduct attacks more efficiently. (Context: Core focus of advanced Red Teaming.)

    Adware: A type of legitimate software that is sometimes used to display digital advertisements in applications. (Context: Can sometimes be used maliciously to track user behavior or serve as a backdoor.)

    Algorithm: A set of rules used to solve a problem. (Context: The mathematical foundation behind encryption protocols and AI models.)

    Analysis: The investigation and validation of alerts. (Context: The daily operational duty of a SOC analyst.)

    Angler phishing: A technique where attackers impersonate customer service representatives on social media. (Context: Exploits user frustration to steal account details.)

    Anomaly-based analysis: A detection method that identifies abnormal behavior. (Context: Catches zero-day attacks by comparing traffic to a known "normal" baseline.)

    Antivirus software: A software program used to prevent, detect, and eliminate malware and viruses. (Context: A standard endpoint control.)

    Application: A program that performs a specific task. (Context: Applications are often the target of attacks like XSS or SQL injection.)

    Application programming interface (API) token: A small block of encrypted code that contains information about a user. (Context: Used for secure authentication between systems without sending raw credentials.)

    Argument (Linux): Specific information needed by a command. (Context: Used to specify the target of a command, like a file name or IP address.)

    Argument (Python): The data brought into a function when it is called. (Context: Allows functions to be dynamic and process different data inputs.)

    Array: A data type that stores data in a comma-separated ordered list. (Context: Used to store multiple items of the same data type sequentially.)

    Assess: The fifth step of the NIST RMF that means to determine if established controls are implemented correctly. (Context: Crucial for ensuring that security designs actually work in practice, not just in theory.)

    Asset: An item perceived as having value to an organization. (Context: The specific targets that security frameworks and access controls are designed to protect.)

    Asset classification: The practice of labeling assets based on sensitivity and importance to an organization. (Context: Helps prioritize security efforts on the most critical assets.)

    Asset inventory: A catalog of assets that need to be protected. (Context: The foundational step of understanding what assets you actually have on your network.)

    Asset management: The process of tracking assets and the risks that could affect them. (Context: Ensures all assets are accounted for and secured.)

    Asymmetric encryption: The use of a public and private key pair for encryption and decryption of data. (Context: Used for secure key exchange and digital signatures.)

    Attack surface: All the potential vulnerabilities that a threat actor could exploit. (Context: Organizations try to minimize their attack surface to reduce risk.)

    Attack tree: A diagram that maps threats to assets. (Context: Used to visually understand how an attacker might compromise an asset.)

    Attack vectors: The pathways attackers use to penetrate security defenses. (Context: Understanding these helps analysts prioritize where to place preventative controls.)

    Authentication: The process of verifying who someone is. (Context: The first step in IAM, often enforced via MFA or biometrics.)

    Authorization: The concept of granting access to specific resources in a system. (Context: Follows authentication; enforced via the Principle of Least Privilege.)

    Authorize: The sixth step of the NIST RMF that refers to being accountable for the security and privacy risks that might exist in an organization. (Context: Requires official sign-off from management before a system goes live.)

    Automation: The use of technology to reduce human and manual effort to perform common and repetitive tasks. (Context: Helps reduce the risk of human error and frees up analysts to focus on more complex tasks.)

    Availability: The idea that data is accessible to those who are authorized to access it. (Context: The 'A' in the CIA triad, protected by redundant servers and backups.)

Section B

    Baiting: A social engineering tactic that tempts people into compromising their security. (Context: Exploit human curiosity or greed.)

    Bandwidth: The maximum data transmission capacity over a network, measured by bits per second. (Context: DoS attacks often aim to exhaust a network's bandwidth.)

    Baseline configuration (baseline image): A documented set of specifications within a system that is used as a basis for future builds, releases, and updates. (Context: Ensures consistency and security across multiple systems.)

    Bash: The default shell in most Linux distributions. (Context: The primary interface for interacting with the Linux operating system via the command line.)

    Basic auth: The technology used to establish a user’s request to access a server. (Context: Often involves sending a username and password encoded in base64.)

    Basic Input/Output System (BIOS): A microchip that contains loading instructions for the computer and is prevalent in older systems. (Context: Being replaced by UEFI in modern systems.)

    Biometrics: The unique physical characteristics that can be used to verify a person’s identity. (Context: A "Something you are" authentication factor, highly resilient against brute-force attacks.)

    Bit: The smallest unit of data measurement on a computer. (Context: Data is stored and processed as a series of bits (0s and 1s).)

    Boolean data: Data that can only be one of two values: either True or False. (Context: Used in programming to make decisions based on conditions.)

    Bootloader: A software program that boots the operating system. (Context: The first program that runs when a computer starts up.)

    Botnet: A collection of computers infected by malware that are under the control of a single threat actor, known as the “bot-herder". (Context: Frequently used to launch massive Distributed Denial of Service (DDoS) attacks.)

    Bracket notation: The indices placed in square brackets. (Context: Used in Python to access specific elements within a list or string.)

    Broken chain of custody: Inconsistencies in the collection and logging of evidence in the chain of custody. (Context: Can render evidence inadmissible in a court of law.)

    Brute force attack: The trial and error process of discovering private information. (Context: Mitigated by implementing account lockout policies.)

    Bug bounty: Programs that encourage freelance hackers to find and report vulnerabilities. (Context: A proactive way for organizations to identify and fix security flaws.)

    Built-in function: A function that exists within Python and can be called directly. (Context: Functions like print() or type() that are always available.)

    Business continuity: An organization's ability to maintain their everyday productivity by establishing risk disaster recovery plans. (Context: Directly supports the Availability pillar of the CIA Triad.)

    Business continuity plan (BCP): A document that outlines the procedures to sustain business operations during and after a significant disruption. (Context: Essential for ensuring an organization can survive a major cyberattack or natural disaster.)

    Business Email Compromise (BEC): A type of phishing attack where a threat actor impersonates a known source to obtain financial advantage. (Context: Usually designed to redirect wire transfers or steal financial data.)

Section C

    Categorize: The second step of the NIST RMF that is used to develop risk management processes and tasks. (Context: Helps prioritize systems based on the sensitivity of the data they hold.)

    CentOS: An open-source distribution that is closely related to Red Hat. (Context: A popular Linux distribution for servers.)

    Central Processing Unit (CPU): A computer’s main processor, which is used to perform general computing tasks on a computer. (Context: The "brain" of the computer.)

    Chain of custody: The process of documenting evidence possession and control during an incident lifecycle. (Context: Ensures evidence remains legally admissible.)

    Chronicle: A cloud-native tool designed to retain, analyze, and search data. (Context: Google’s proprietary SIEM tool used for rapid threat hunting across massive datasets.)

    Cipher: An algorithm that encrypts information. (Context: Uses mathematical operations to convert plaintext into ciphertext.)

    Cloud-based firewalls: Software firewalls that are hosted by the cloud service provider. (Context: Protects cloud infrastructure from unauthorized access.)

    Cloud computing: The practice of using remote servers, applications, and network services that are hosted on the internet instead of on local physical devices. (Context: Introduces new security challenges, such as securing APIs and managing access controls in the cloud.)

    Cloud network: A collection of servers or computers that stores resources and data in remote data centers that can be accessed via the internet. (Context: The infrastructure that supports cloud computing.)

    Cloud security: The process of ensuring that assets stored in the cloud are properly configured and access to those assets is limited to authorized users. (Context: A rapidly growing subfield as enterprises migrate away from physical data centers.)

    Command: An instruction telling the computer to do something. (Context: Used in the command line interface to interact with the operating system.)

    Command and control (C2): The techniques used by malicious actors to maintain communications with compromised systems. (Context: Attackers use C2 servers to send instructions to malware infected systems.)

    Command-line interface (CLI): A text-based user interface that uses commands to interact with the computer. (Context: Preferred by many security professionals for its speed and power.)

    Comment: A note programmers make about the intention behind their code. (Context: Helps explain the logic of the code to others or to themselves later.)

    Common Event Format (CEF): A log format that uses key-value pairs to structure data and identify fields and their corresponding values. (Context: A standard format for logging security events.)

    Common Vulnerabilities and Exposures (CVE®) list: An openly accessible dictionary of known vulnerabilities and exposures. (Context: Provides a standardized way to identify and reference security vulnerabilities.)

    Common Vulnerability Scoring System (CVSS): A measurement system that scores the severity of a vulnerability. (Context: Helps organizations prioritize which vulnerabilities to patch first based on their potential impact.)

    Compliance: The process of adhering to internal standards and external regulations. (Context: Essential to avoid massive fines from frameworks like GDPR or HIPAA.)

    Computer security incident response teams (CSIRT): A specialized group of security professionals that are trained in incident management and response. (Context: Initially created in response to the devastating Morris Worm in 1988.)

    Computer virus: Malicious code written to interfere with computer operations and cause damage to data and software. (Context: Typically requires user interaction to execute and spread.)

    Conditional statement: A statement that evaluates code to determine if it meets a specified set of conditions. (Context: Used in programming to execute different blocks of code based on whether a condition is true or false.)

    Confidentiality: The idea that only authorized users can access specific assets or data. (Context: The 'C' in the CIA triad, enforced via encryption and strong IAM policies.)

    Confidential data: Data that often has limits on the number of people who have access to it. (Context: Includes PII, financial data, and trade secrets.)

    Confidentiality, integrity, availability (CIA) triad: A model that helps inform how organizations consider risk when setting up systems and security policies. (Context: The absolute foundation of all security architecture.)

    Configuration file: A file used to configure the settings of an application. (Context: Misconfigured files can introduce security vulnerabilities.)

    Containment: The act of limiting and preventing additional damage caused by an incident. (Context: A crucial step in the incident response process to prevent the attacker from spreading further.)

    Controlled zone: A subnet that protects the internal network from the uncontrolled zone. (Context: A network segment with stricter security controls.)

    Cross-site scripting (XSS): An injection attack that inserts code into a vulnerable website or web application. (Context: Can be used to steal session cookies or redirect users to malicious sites.)

    Crowdsourcing: The practice of gathering information using public input and collaboration. (Context: Used in bug bounty programs to find vulnerabilities.)

    Cryptographic attack: An attack that affects secure forms of communication between a sender and intended recipient. (Context: Aims to decipher encrypted data without the key.)

    Cryptographic key: A mechanism that decrypts ciphertext. (Context: Essential for accessing encrypted data.)

    Cryptography: The process of transforming information into a form that unintended readers can’t understand. (Context: Relies on complex mathematical algorithms to encrypt and decrypt data.)

    Cryptojacking: A form of malware that installs software to illegally mine cryptocurrencies. (Context: Steals computing resources and can significantly degrade system performance.)

    CVE Numbering Authority (CNA): An organization that volunteers to analyze and distribute information on eligible CVEs. (Context: Responsible for assigning CVE IDs to newly discovered vulnerabilities.)

    Cybersecurity (or security): The practice of ensuring confidentiality, integrity, and availability of information by protecting networks, devices, people, and data from unauthorized access or criminal exploitation. (Context: The overarching field encompassing all these concepts.)

Section D

    Data: Information that is translated, processed, or stored by a computer. (Context: The primary asset that security professionals seek to protect.)

    Data at rest: Data not currently being accessed. (Context: Protected primarily through encryption.)

    Database: An organized collection of information or data. (Context: Often the target of SQL injection attacks.)

    Data controller: A person that determines the procedure and purpose for processing data. (Context: A key role defined under GDPR.)

    Data custodian: Anyone or anything that’s responsible for the safe handling, transport, and storage of information. (Context: Responsible for implementing the security controls defined by the data owner.)

    Data exfiltration: Unauthorized transmission of data from a system. (Context: Often the ultimate goal of a cyberattack.)

    Data in transit: Data traveling from one point to another. (Context: Protected primarily through encryption protocols like TLS.)

    Data in use: Data being accessed by one or more users. (Context: The most difficult state to protect, often relying on access controls and endpoint security.)

    Data owner: The person who decides who can access, edit, use, or destroy their information. (Context: Ultimately responsible for the security of the data.)

    Data packet: A basic unit of information that travels from one device to another within a network. (Context: Analyzed by IDS/IPS and packet sniffers to identify malicious traffic.)

    Data point: A specific piece of information. (Context: Individual elements within a database or dataset.)

    Data processor: A person that is responsible for processing data on behalf of the data controller. (Context: A key role defined under GDPR.)

    Data protection officer (DPO): An individual that is responsible for monitoring the compliance of an organization's data protection procedures. (Context: Required by GDPR for certain organizations.)

    Data type: A category for a particular type of data item. (Context: In programming, defines what operations can be performed on the data (e.g., integer, string, boolean).)

    Date and time data: Data representing a date and/or time. (Context: Crucial in log analysis for establishing the timeline of an incident.)

    Debugger: A software tool that helps to locate the source of an error and assess its causes. (Context: Used by programmers and analysts to identify flaws in code or scripts.)

    Debugging: The practice of identifying and fixing errors in code. (Context: Essential for ensuring scripts and automation tools function correctly.)

    Defense in depth: A layered approach to vulnerability management that reduces risk. (Context: Employs multiple security controls (e.g., firewalls, IDS, endpoint security) to protect against threats.)

    Denial of service (DoS) attack: An attack that targets a network or server and floods it with network traffic. (Context: Aims to disrupt the availability of a system or service.)

    Detect: A NIST core function related to identifying potential security incidents and improving monitoring capabilities to increase the speed and efficiency of detections. (Context: Involves deploying IDS, EDR, and SIEM alerts to catch threats quickly.)

    Detection: The prompt discovery of security events. (Context: The goal of the Detect function in the NIST framework.)

    Dictionary data: Data that consists of one or more key-value pairs. (Context: A data structure used in Python to store data in a way that allows for fast retrieval.)

    Digital certificate: A file that verifies the identity of a public key holder. (Context: Used in PKI to establish trust and enable secure communication (e.g., HTTPS).)

    Digital forensics: The practice of collecting and analyzing data to determine what has happened after an attack. (Context: Critical for post-incident investigations and maintaining a strict chain of custody.)

    Directory: A file that organizes where other files are stored. (Context: Also known as a folder in some operating systems.)

    Disaster recovery plan: A plan that allows an organization’s security team to outline the steps needed to minimize the impact of a security incident. (Context: Focuses on restoring IT infrastructure and operations after a disruption.)

    Distributed denial of service (DDoS) attack: A type of denial or service attack that uses multiple devices or servers located in different locations to flood the target network with unwanted traffic. (Context: Often utilizes botnets to generate massive amounts of traffic.)

    Distributions: The different versions of Linux. (Context: Examples include Ubuntu, CentOS, and Kali Linux.)

    Documentation: Any form of recorded content that is used for a specific purpose. (Context: Essential for incident response, compliance, and maintaining system configurations.)

    DOM-based XSS attack: An instance when malicious script exists in the webpage a browser loads. (Context: A specific type of Cross-Site Scripting attack that exploits vulnerabilities in the Document Object Model.)

    Domain Name System (DNS): A networking protocol that translates internet domain names into IP addresses. (Context: The "phonebook" of the internet, often targeted by attackers (e.g., DNS spoofing).)

    Dropper: A program or a file used to install a rootkit on a target computer. (Context: A common technique used by malware to establish a persistent presence on a system.)

Section E

    Elevator pitch: A brief summary of your experience, skills, and background. (Context: Used to quickly introduce yourself to potential employers or networking contacts.)

    Encapsulation: A process performed by a VPN service that protects your data by wrapping sensitive data in other data packets. (Context: Enhances privacy and security when transmitting data over untrusted networks.)

    Encryption: The process of converting data from a readable format to an encoded format. (Context: The primary mechanism for protecting data at rest and data in transit.)

    Endpoint: Any device connected on a network. (Context: Examples include laptops, smartphones, and servers; often the primary target of attacks.)

    Endpoint detection and response (EDR): An application that monitors an endpoint for malicious activity. (Context: A major upgrade from traditional antivirus, relying on behavioral analysis.)

    Eradication: The complete removal of the incident elements from all affected systems. (Context: A key phase in the incident response lifecycle.)

    Escalation policy: A set of actions that outline who should be notified when an incident alert occurs and how that incident should be handled. (Context: Ensures incidents are addressed by the appropriate personnel in a timely manner.)

    Event: An observable occurrence on a network, system, or device. (Context: Not all events are incidents, but all incidents are composed of events.)

    Exception: An error that involves code that cannot be executed even though it is syntactically correct. (Context: Often occurs during runtime due to unexpected input or conditions.)

    Exclusive operator: An operator that does not include the value of comparison. (Context: Examples include > (greater than) and < (less than).)

    Exploit: A way of taking advantage of a vulnerability. (Context: Attackers use exploits to gain unauthorized access or cause harm.)

    Exposure: A mistake that can be exploited by a threat. (Context: Often a misconfiguration or an unpatched vulnerability.)

    External threat: Anything outside the organization that has the potential to harm organizational assets. (Context: Includes APTs, hacktivists, and opportunistic cybercriminals.)

Section F

    False negative: A state where the presence of a threat is not detected. (Context: A critical failure of a security system, as a real threat goes unnoticed.)

    False positive: An alert that incorrectly detects the presence of a threat. (Context: Can cause "alert fatigue" for security analysts if too frequent.)

    Fileless malware: Malware that does not need to be installed by the user because it uses legitimate programs that are already installed to infect a computer. (Context: Highly evasive because it doesn't leave a traditional footprint on the hard drive.)

    File path: The location of a file or directory. (Context: Used to navigate the file system in Linux and other operating systems.)

    Filesystem Hierarchy Standard (FHS): The component of the Linux OS that organizes data. (Context: Defines the standard directory structure and its contents in Linux distributions.)

    Filtering: Selecting data that match a certain condition. (Context: A common operation in SIEM tools and SQL databases to find relevant information.)

    Final report: Documentation that provides a comprehensive review of an incident. (Context: Crucial for understanding what happened, how it was handled, and how to improve future responses.)

    Firewall: A network security device that monitors traffic to or from a network. (Context: Acts as the primary barrier between a trusted internal network and an untrusted external network.)

    Float data: Data consisting of a number with a decimal point. (Context: A specific data type used in programming.)

    Foreign key: A column in a table that is a primary key in another table. (Context: Used in relational databases to link tables together.)

    Forward proxy server: A server that regulates and restricts a person’s access to the internet. (Context: Can be used to filter content or improve performance through caching.)

    Function: A section of code that can be reused in a program. (Context: Promotes code reusability and organization in programming.)

Section G

    Global variable: A variable that is available through the entire program. (Context: Can be accessed from any function or block of code.)

    Graphical user interface (GUI): A user interface that uses icons on the screen to manage different tasks on the computer. (Context: More user-friendly than a CLI, but often less powerful for complex tasks.)

Section H

    Hacker: Any person or group who uses computers to gain unauthorized access to data. (Context: Often used synonymously with threat actor.)

    Hacktivist: A person who uses hacking to achieve a political goal. (Context: Motivated by ideology rather than financial gain.)

    Hard drive: A hardware component used for long-term memory. (Context: Where data is stored persistently.)

    Hardware: The physical components of a computer. (Context: Includes the CPU, RAM, hard drive, and network interface card.)

    Hash collision: An instance when different inputs produce the same hash value. (Context: A vulnerability in hash functions that can be exploited by attackers.)

    Hash function: An algorithm that produces a code that can’t be decrypted. (Context: Used to verify data integrity and securely store passwords.)

    Hash table: A data structure that's used to store and reference hash values. (Context: Allows for very fast data retrieval based on keys.)

    Health Insurance Portability and Accountability Act (HIPAA): A U.S. federal law established to protect patients’ health information. (Context: Imposes strict security and privacy requirements on healthcare organizations.)

    Honeypot: A system or resource created as a decoy vulnerable to attacks with the purpose of attracting potential intruders. (Context: Used to study attacker behavior and gather threat intelligence.)

    Host-based intrusion detection system (HIDS): An application that monitors the activity of the host on which it’s installed. (Context: Analyzes system logs, file modifications, and process activity.)

    Hub: A network device that broadcasts information to every device on the network. (Context: Inefficient and insecure compared to switches; largely obsolete in modern networks.)

    Hypertext Transfer Protocol (HTTP): An application layer protocol that provides a method of communication between clients and website servers. (Context: Transmits data in plaintext, making it vulnerable to interception.)

    Hypertext Transfer Protocol Secure (HTTPS): A network protocol that provides a secure method of communication between clients and website servers. (Context: Uses TLS/SSL to encrypt HTTP traffic, ensuring confidentiality and integrity.)

Section I

    Identify: A NIST core function related to management of cybersecurity risk and its effect on an organization’s people and assets. (Context: The foundational step of understanding what assets you actually have on your network.)

    Identity and access management (IAM): A collection of processes and technologies that helps organizations manage digital identities in their environment. (Context: Ensures the right people have the right access to the right tools.)

    IEEE 802.11 (Wi-Fi): A set of standards that define communication for wireless LANs. (Context: The technology that underpins Wi-Fi networks.)

    Immutable: An object that cannot be changed after it is created and assigned a value. (Context: Important for ensuring data integrity, especially in functional programming and certain data structures like tuples.)

    Implement: The fourth step of the NIST RMF that means to implement security and privacy plans for an organization. (Context: Transitioning from theoretical policy to technical reality.)

    Improper usage: An incident type that occurs when an employee of an organization violates the organization’s acceptable use policies. (Context: An example of an internal threat, whether intentional or accidental.)

    Incident: An occurrence that actually or imminently jeopardizes, without lawful authority, the confidentiality, integrity, or availability of information or an information system; or constitutes a violation or imminent threat of violation of law, security policies, security procedures, or acceptable use policies. (Context: A security event that has a negative impact.)

    Incident escalation: The process of identifying a potential security incident, triaging it, and handing it off to a more experienced team member. (Context: Ensures incidents are handled by personnel with the appropriate expertise.)

    Incident handler’s journal: A form of documentation used in incident response. (Context: A detailed log of all actions taken and observations made during an investigation.)

    Incident response: An organization’s quick attempt to identify an attack, contain the damage, and correct the effects of a security breach. (Context: Prevents panic by providing a strict playbook roadmap.)

    Incident response plan: A document that outlines the procedures to take in each step of incident response. (Context: The playbook that guides the organization through a security incident.)

    Inclusive operator: An operator that includes the value of comparison. (Context: Examples include >= (greater than or equal to) and <= (less than or equal to).)

    Indentation: Space added at the beginning of a line of code. (Context: Crucial in Python for defining code blocks and structure.)

    Index: A number assigned to every element in a sequence that indicates its position. (Context: Used to access specific items in an array, list, or string.)

    Indicators of attack (IoA): The series of observed events that indicate a real-time incident. (Context: Focuses on the intent and techniques of the attacker before the attack is complete.)

    Indicators of compromise (IoC): Observable evidence that suggests signs of a potential security incident. (Context: Technical artifacts like IP addresses, file hashes, or URLs that indicate a system has been breached.)

    Information privacy: The protection of unauthorized access and distribution of data. (Context: Closely related to data confidentiality and compliance regulations.)

    Information security (InfoSec): The practice of keeping data in all states away from unauthorized users. (Context: Synonymous with cybersecurity, focusing on protecting the CIA triad of information.)

    Injection attack: Malicious code inserted into a vulnerable application. (Context: Examples include SQL injection and Cross-Site Scripting (XSS).)

    Input validation: Programming that validates inputs from users and other programs. (Context: A critical defense against injection attacks by ensuring data conforms to expected formats.)

    Integer data: Data consisting of a number that does not include a decimal point. (Context: A specific data type used in programming.)

    Integrated development environment (IDE): A software application for writing code that provides editing assistance and error correction tools. (Context: Tools like VS Code or PyCharm that enhance programmer productivity.)

    Integrity: The idea that the data is correct, authentic, and reliable. (Context: The 'I' in the CIA Triad, protected by file hashes and digital signatures.)

    Internal hardware: The components required to run the computer. (Context: E.g., CPU, motherboard, RAM.)

    Internal threat: A current or former employee, external vendor, or trusted partner who poses a security risk. (Context: Highly dangerous because they bypass external perimeter defenses; can be malicious or accidental.)

    Internet Control Message Protocol (ICMP): An internet protocol used by devices to tell each other about data transmission errors across the network. (Context: Often used for diagnostic tools like 'ping', but can also be abused in attacks.)

    Internet Control Message Protocol flood (ICMP flood): A type of DoS attack performed by an attacker repeatedly sending ICMP request packets to a network server. (Context: Aims to overwhelm the server's resources, causing it to become unresponsive.)

    Internet Protocol (IP): A set of standards used for routing and addressing data packets as they travel between devices on a network. (Context: The fundamental protocol that enables the internet.)

    Internet Protocol (IP) address: A unique string of characters that identifies the location of a device on the internet. (Context: Essential for routing data to the correct destination.)

    Interpreter: A computer program that translates Python code into runnable instructions line by line. (Context: Allows Python scripts to be executed without pre-compilation.)

    Intrusion detection system (IDS): An application that monitors system activity and alerts on possible intrusions. (Context: A passive defense tool; it flags the threat but does not actively stop it.)

    Intrusion prevention system (IPS): An application that monitors system activity for intrusive activity and takes action to stop the activity. (Context: An active defense tool that can block malicious traffic in real-time.)

    IP spoofing: A network attack performed when an attacker changes the source IP of a data packet to impersonate an authorized system and gain access to a network. (Context: Used to bypass IP-based access controls or hide the attacker's identity.)

    Iterative statement: Code that repeatedly executes a set of instructions. (Context: Often implemented using 'for' or 'while' loops in programming.)

Section K

    KALI LINUX ™: An open-source distribution of Linux that is widely used in the security industry. (Context: Comes pre-installed with numerous penetration testing and security auditing tools.)

    Kernel: The component of the Linux OS that manages processes and memory. (Context: The core of the operating system, acting as a bridge between applications and hardware.)

    Key-value pair: A set of data that represents two linked items: a key, and its corresponding value. (Context: Used in dictionaries (Python) and JSON to structure data.)

Section L

    Legacy operating system: An operating system that is outdated but still being used. (Context: Presents a significant security risk as it may no longer receive security updates.)

    Lessons learned meeting: A meeting that includes all involved parties after a major incident. (Context: A critical part of the post-incident activity phase to improve future responses.)

    Library: A collection of modules that provide code users can access in their programs. (Context: Provides pre-written functions to save time and effort in programming.)

    Linux: An open-source operating system. (Context: Widely used for servers, embedded systems, and security tools.)

    List concatenation: The concept of combining two lists into one by placing the elements of the second list directly after the elements of the first list. (Context: A common operation when manipulating data in Python.)

    List data: Data structure that consists of a collection of data in sequential form. (Context: Allows storing multiple items in a single variable in Python.)

    Loader: Malicious code that launches after a user initiates a dropper program. (Context: Responsible for executing the primary payload of the malware.)

    Local Area Network (LAN): A network that spans small areas like an office building, a school, or a home. (Context: A privately owned and controlled network.)

    Local variable: A variable assigned within a function. (Context: Only accessible within the scope of that specific function.)

    Log: A record of events that occur within an organization’s systems. (Context: The foundational raw data analyzed by SIEM tools to find anomalies.)

    Log analysis: The process of examining logs to identify events of interest. (Context: Crucial for detecting security incidents and troubleshooting system issues.)

    Logging: The recording of events occurring on computer systems and networks. (Context: The process of generating log files.)

    Logic error: An error that results when the logic used in code produces unintended results. (Context: The code runs without crashing, but the output is incorrect.)

    Log management: The process of collecting, storing, analyzing, and disposing of log data. (Context: Essential for maintaining a secure and auditable environment.)

    Loop condition: The part of a loop that determines when the loop terminates. (Context: Ensures a loop doesn't run infinitely.)

    Loop variable: A variable that is used to control the iterations of a loop. (Context: Often used as a counter or an index within the loop.)

Section M

    Malware: Software designed to harm devices or networks. (Context: Includes sub-categories like viruses, worms, and ransomware.)

    Malware infection: An incident type that occurs when malicious software designed to disrupt a system infiltrates an organization’s computers or network. (Context: Often the result of phishing or exploiting software vulnerabilities.)

    Media Access Control (MAC) address: A unique alphanumeric identifier that is assigned to each physical device on a network. (Context: Used for communication within a local network segment.)

    Method: A function that belongs to a specific data type. (Context: In object-oriented programming, a method is a function associated with an object.)

    Metrics: Key technical attributes such as response time, availability, and failure rate, which are used to assess the performance of a software application. (Context: Used to evaluate the effectiveness of security controls and overall system health.)

    MITRE: A collection of non-profit research and development centers. (Context: Known for maintaining the CVE list and the ATT&CK framework.)

    Modem: A device that connects your router to the internet and brings internet access to the LAN. (Context: Modulates and demodulates signals for transmission over communication lines.)

    Watering hole attack: A type of attack when a threat actor compromises a website frequently visited by a specific group of users. (Context: Highly effective against localized organizations or specific industries.)

    Web-based exploits: Malicious code or behavior that’s used to take advantage of coding flaws in a web application. (Context: Exploits like Cross-Site Scripting (XSS) or SQL Injection.)

    Whaling: A category of spear phishing attempts that are aimed at high-ranking executives in an organization. (Context: Used to gain access to highly sensitive, restricted corporate data.)

    Wide Area Network (WAN): A network that spans a large geographic area like a city, state, or country. (Context: The internet itself is the largest WAN.)

    Wi-Fi Protected Access (WPA): A wireless security protocol for devices to connect to the internet. (Context: Replaced the heavily flawed WEP protocol.)

    Wildcard: A special character that can be substituted with any other character. (Context: Heavily used in Linux terminal commands and SQL scripting to batch process files.)

    Wireshark: An open-source network protocol analyzer. (Context: The industry standard tool for deep packet inspection and network troubleshooting.)

    World-writable file: A file that can be altered by anyone in the world. (Context: A massive security risk in Linux environments; usually flagged during security audits.)

    Worm: Malware that can duplicate and spread itself across systems on its own. (Context: The Blaster and Morris worms are historical examples of how devastating these can be to networks.)

    YARA-L: A computer language used to create rules for searching through ingested log data. (Context: Specific to modern SIEM tools like Google Chronicle for threat hunting.)

    Zero-day: An exploit that was previously unknown. (Context: Highly dangerous because no official patch exists when the attack occurs in the wild.)
