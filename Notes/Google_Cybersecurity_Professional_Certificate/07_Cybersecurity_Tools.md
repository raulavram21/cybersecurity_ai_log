Cybersecurity Tools and Programing Languages


Logs and SIEM Tools

    Log: A record of events that occur within an organization’s systems. (Context: The foundational source of data that security tools organize and analyze to detect anomalies.)

    SIEM (Security Information and Event Management): An application that collects and analyzes log data to monitor critical activities in real-time. (Context: Reduces the massive amount of raw data an analyst must review by automatically generating alerts for specific threats and organizing data into visual dashboards.)

    Splunk / Splunk Enterprise: A self-hosted data analysis platform that provides SIEM solutions to retain, analyze, and search an organization's log data.

    Chronicle: A cloud-native SIEM tool used to store security data for rapid search and analysis. (Context: Cloud-hosted options like Chronicle are often easier for teams to set up and maintain compared to on-premise solutions.)

Network Protocol Analyzers

    Network Protocol Analyzer (Packet Sniffer): A tool designed to capture, record, and analyze data traffic moving across a network. (Context: Tools like Wireshark and tcpdump allow analysts to deeply inspect the exact packets of data entering and leaving their systems to identify malicious activity.)

Playbooks and Digital Forensics

    Playbook: A step-by-step manual that provides details about operational actions, such as how to properly respond to a security incident. (Context: Ensures teams follow standardized, legally sound protocols during high-stress situations like a data breach.)

    Chain of Custody: The rigorous process of documenting evidence possession and control during an incident lifecycle. (Context: Ensures evidence remains legally admissible by recording exactly who, what, where, and why evidence is being held or moved at all times.)

    Protecting and Preserving Evidence: The procedural process of properly handling fragile and volatile digital evidence. (Context: To prevent compromising original evidence during an investigation, analysts must preserve the data by making copies and conducting the investigation exclusively on those copies.)

    Order of Volatility: A precise sequence outlining the order in which data must be preserved from first to last. (Context: Prioritizes capturing volatile data—like active system memory—which will be lost permanently if the compromised device powers off.)
